## Directory Purposes

**src/components/ui/:**
- Purpose: Radix UI primitive wrappers (button, dialog, card, select, etc.) auto-generated by shadcn CLI
- Contains: Re-exported shadcn components with Tailwind styling
- Key files: 200+ component files (do not edit; use `shadcn-ui add <component>` to update)

**src/components/adaptation/:**
- Purpose: Multi-step activity adaptation wizard and result rendering
- Contains: Step components (StepActivityType, StepActivityInput, StepBarrierSelection, StepResult, StepExport), renderers, edit modal
- Key files: `AdaptationWizard.tsx` (5-step orchestrator), `StructuredContentRenderer.tsx` (renders typed JSON), `StepResult.tsx` (generates AI result)

**src/components/admin/:**
- Purpose: Admin-only dashboard and reporting
- Contains: Teacher management list, AI usage analytics
- Key files: `TeacherManagement.tsx`, `ai-usage/` subdirectory with charts

**src/components/chat/:**
- Purpose: AI chat interface (separate from adaptation wizard)
- Contains: Chat message display, input, streaming response handler

**src/pages/:**
- Purpose: Top-level route components
- Contains: Dashboard, Login, MyAdaptations, QuestionBank, Classes, Settings, Chat, etc.
- Key files: `Create.tsx` (large, 773 lines; free-form activity creation), `MyAdaptations.tsx` (adaptation history), `QuestionBank.tsx` (question management, 1236 lines)

**src/pages/admin/:**
- Purpose: Admin-only pages
- Contains: TeacherManagement, AiUsageReport (routes accessible only with admin role)

**src/lib/pdf/:**
- Purpose: React PDF renderer for adaptation export
- Contains: PDF page components (header, content, footer), HTML-to-element parser, LaTeX/math parser, font registration
- Key files: `index.tsx` (main Document component), `htmlToPdfElements.ts` (HTML conversion), `textParser.ts` (LaTeX fractions)

**src/lib/:**
- Purpose: Reusable business logic, transformations, integrations
- Contains: Export (PDF/DOCX), AI streaming, validation, data format conversion, parsing utilities
- Key files: `exportDocx.ts` (452 lines; handles fractions, images, layouts), `streamAI.ts` (SSE parsing), `barriers.ts` (domain data)

**src/types/:**
- Purpose: TypeScript type definitions for core domain
- Contains: `adaptation.ts` with `StructuredActivity`, `StructuredQuestion`, `Alternative`, type guards
- Used by: All components, services, validation

**src/integrations/supabase/:**
- Purpose: Database client and auto-generated schema types
- Contains: `client.ts` (singleton, auto-generated), `types.ts` (Database type from schema)
- Critical: Do not manually edit; regenerate with `supabase gen types` when schema changes

**src/hooks/:**
- Purpose: React hooks for shared logic, auth, data fetching, device detection
- Contains: `useAuth()` context, custom query hooks, media query hooks
- Key files: `useAuth.tsx` (authentication state machine)

**src/test/:**
- Purpose: Test files mirroring src structure
- Contains: Unit tests (lib functions, utilities), integration tests (multi-component flows), component tests
- Patterns: Vitest + Testing Library, helpers for mocking Supabase, fixtures for test data
- Key files: `setup.ts` (Vitest config), `helpers.ts` (createTestWrapper, createSupabaseMock), `fixtures.ts` (reusable test data)

## Key File Locations

**Entry Points:**
- `src/main.tsx`: React app mount point
- `src/App.tsx`: Root component with router and providers
- `src/vite-env.d.ts`: Vite client type definitions

**Configuration:**
- `vite.config.ts`: Vite build config (path alias `@`, plugins, optimizeDeps)
- `tsconfig.json`: TypeScript compiler options (strict checks disabled for flexibility)
- `.prettierrc`, `.eslintrc.*`: Code style enforcement (if present)

**Core Logic:**
- `src/components/adaptation/AdaptationWizard.tsx`: 5-step wizard state machine
- `src/lib/streamAI.ts`: Server-sent event parsing for AI responses
- `src/lib/exportDocx.ts`: DOCX generation with math support
- `src/lib/adaptedQuestions.ts`: Parse/replace question numbers in adapted text
- `src/types/adaptation.ts`: Core domain types

**Testing:**
- `src/test/setup.ts`: Vitest configuration
- `src/test/helpers.ts`: Mock functions and test wrappers
- `src/test/fixtures.ts`: Reusable test data (activities, barriers, etc.)
- `src/test/integration-*.test.tsx`: Multi-component user flow tests

## Naming Conventions

**Files:**
- Components: PascalCase (`AdaptationWizard.tsx`, `Layout.tsx`)
- Hooks: camelCase with `use` prefix (`useAuth.tsx`, `use-toast.ts`)
- Utilities/services: camelCase (`streamAI.ts`, `exportDocx.ts`, `barriers.ts`)
- Tests: MatchesSource.test.ts or integration-FlowName.test.tsx
- Types/interfaces: PascalCase, optional `.types.ts` suffix (but inline in `adaptation.ts`)

**Directories:**
- Feature folders: lowercase, descriptive (`adaptation`, `chat`, `admin`, `ai-usage`)
- Utility folders: lowercase, plural (`components`, `hooks`, `lib`, `pages`, `types`, `integrations`)

**Code Inside Files:**
- Components: PascalCase (`AdaptationWizard`, `StepActivityType`)
- Functions: camelCase (`streamAI`, `exportToPdf`, `buildQuestionImageMap`)
- Types: PascalCase (`StructuredActivity`, `WizardData`, `BarrierItem`)
- Constants: SCREAMING_SNAKE_CASE (`STEPS`, `QUESTION_TYPE_LABELS`)
- React hooks: camelCase with `use` prefix (`useAuth`, `useQuery`)

## Where to Add New Code

**New Feature (e.g., Barrier-Specific Adaptations):**
- Primary logic: Create file in `src/lib/` (e.g., `barrierAdaptations.ts`)
- Components: Add component file in `src/components/` or sub-folder if related to existing feature
- Types: Add to `src/types/adaptation.ts` or create `src/types/newFeature.ts`
- Tests: Create `src/test/lib/barrierAdaptations.test.ts` and component tests in `src/test/components/`

**New Page:**
- Implementation: `src/pages/NewPageName.tsx`
- Routing: Add route in `src/App.tsx` Routes
- Protection: Wrap with `<ProtectedRoute>` if authenticated-only, or `<AdminRoute>` if admin-only
- Layout: Import and use `<Layout>` as parent layout for dashboard routes
- Tests: Create `src/test/pages/NewPageName.test.tsx`

**New Component:**
- Implementation: `src/components/NewComponent.tsx`
- If feature-specific: `src/components/existingFeature/NewComponent.tsx`
- If reusable UI primitive: `src/components/ui/NewComponent.tsx` (use shadcn CLI first)
- Tests: Create corresponding test file in `src/test/components/`

**Utilities/Helpers:**
- General utilities: `src/lib/utils.ts`
- Domain-specific logic: `src/lib/featureName.ts` (e.g., `barriers.ts`, `streamAI.ts`)
- Export utilities: Keep in `src/lib/exportDocx.ts`, `src/lib/exportPdf.ts`
- Tests: Create `src/test/lib/featureName.test.ts`

**Types & Interfaces:**
- Core adaptation types: `src/types/adaptation.ts`
- Feature-specific types: Inline in files or in `src/types/featureName.ts`
- Database types: Auto-generated in `src/integrations/supabase/types.ts` (do not edit)

**Hooks:**
- React context: `src/hooks/useName.tsx` (includes provider export)
- Data fetching: Inline in component or in `src/hooks/useQueryName.ts` if reused
- DOM utilities: `src/hooks/use-something.tsx` (lowercase with hyphen, shadcn style)

## Special Directories

**src/components/ui/:**
- Purpose: Auto-generated shadcn component library
- Generated: Yes (via `shadcn-ui add`)
- Committed: Yes
- Notes: Do not edit manually; use `shadcn-ui@latest add <component>` to add or update

**src/lib/pdf/**
- Purpose: Complex PDF rendering pipeline
- Generated: No
- Committed: Yes
- Notes: Text parsing (LaTeX, HTML) is fragile; test thoroughly before changes

**src/integrations/supabase/**
- Purpose: Auto-generated database client and types
- Generated: Yes (`supabase gen types`)
- Committed: Yes
- Notes: `client.ts` header warns "Do not edit directly"; types.ts is auto-generated from schema

**src/test/**
- Purpose: Test infrastructure mirroring src structure
- Generated: No (manually created)
- Committed: Yes
- Notes: `setup.ts` configures Vitest, `helpers.ts` provides mocks for Supabase and React Query

**node_modules/.pnpm** / **node_modules/**
- Purpose: Dependency storage
- Generated: Yes (via `bun install` or `npm install`)
- Committed: No (.gitignore)
